Thinking like a programmer

This Nanodegree will focuse on five diffent ways of "think like a computer progammer".

  1. Procedural Thinking

    Creating perfectly clear and unambiguous instruction for a computer to follow.

  2. Abstract Thinking

    Finding similarity or generality amongst seemingly diffrent things.

    Abstraction is used in programming to avoid unnecessary repetition of work.

  3. Systems Thinking

    Breaking a big problem down into smaller manageable pieces.

    It involves big-picture thinking and decision-making about a problem and how different pieces of a program can work together to solve it.

  4. Technological Empathy

    Comes in many forms. For example, computer empathy is the ability to understand what a computer is, how it works, and what it´s good and bad at doing.

    As a programmer it is vital to have a basic understanding how a computer and the tools are working.

  5. Debugging

    Debugging is a systematic process to identifying the cause to way a program doesn´t work.

    A basic systematic process can be:

    1. Collect evidence (what makes this program fail?)
    2. Generate theories (what may have caused this problem?)
    3. Test those theories (if my theory is correct, how could I find out?)
    4. Fix the problem

Basic of the Web and html

World wide Web (WWW) consists of HTML pages that uses HyperText Markup Language (HTML) to markup the pages that the web browser interpets and renders to the user.

HTML pages are linked together with Hyper Links (links).

The web was invented in the early 1990s and have over 30 Billion pages.

Examples of files that can be found on the web:

  • Plain text
  • HTML
  • Images
  • Videos
  • Music

Major Pieses of the web:

  • The user that consist of:
    • Computer
    • Browser
      • A program that runs on the computer to display files found on the web.
  • The Internet
    • The worlds largest computer network.
  • Servers
    • Computers that host the files that make up the web.
  • The browser communicates with the servers over the internet using HTTP.
    • HTTP - the main protocol of the web.

HTML is mad up of:

  • Text content.
  • Markup.
  • References to other documents.
  • Links to other pages.

Inline vs Block

Every HTML element has a default display value depending on what type of element it is. The default display value for most elements is block or inline.

Block-level Elements

A block-level element always starts on a new line and takes up the full width available (stretches out to the left and right as far as it can).

Examples of block-level elements:

  • <div>
  • <h1> - <h6>
  • <p>
  • <form>

Inline Elements

An inline element does not start on a new line and only takes up as much width as necessary.

Examples of block-level elements:

  • <span>
  • <a>
  • <img>
